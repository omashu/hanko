# Hanko

Личное десктопное приложение: раздел «Манга/манхва» ищет и читает тайтлы через открытый
API MangaDex прямо в приложении (ничего не скачивая насовсем на диск), раздел «Аниме» —
это твои сайты, сохранённые как плитки, которые открываются во встроенной вкладке-браузере
внутри самого приложения — без переключения в Chrome.

Название и иконка — рабочий вариант, меняются в одну секунду (см. ниже).

---

## Запуск (для разработки)

Node.js у тебя уже стоит после MythForge, так что:

1. Открой папку `hanko` в VS Code (Файл → Открыть папку…).
2. Терминал → Новый терминал.
3. Один раз (в этот раз обязательно, даже если раньше уже делал — снова появилась
   новая зависимость, теперь для автообновления): `npm install`
4. Дальше просто: `npm start`

## Друзья и чат (онлайн, через Supabase) — разовая настройка

Раздел «Профиль» умеет по-настоящему добавлять друзей (по нику, не по коду) и
переписываться с ними в реальном времени, а также — по желанию — регистрироваться
по почте и паролю, чтобы профиль не был привязан только к этому компьютеру.
Работает через твой проект Supabase — без него раздел просто покажет
«Онлайн недоступен». Настройка — один раз (или ещё раз, если уже делал раньше:
в схему добавился ник — файл безопасно перезапускать):

1. **Включи анонимные входы.** В Supabase Dashboard → Authentication → Sign In / Providers →
   найди **Anonymous Sign-Ins** → включи. Без этого шага у приложения не получится
   создать даже "невидимого" пользователя для твоего компьютера.
2. **Выполни SQL-схему.** В Supabase Dashboard → SQL Editor → New query → вставь целиком
   содержимое файла `supabase_schema.sql` (лежит в этой же папке) → Run. Это создаёт
   таблицы (`profiles`, `friend_requests`, `friends`, `messages`), правила доступа (RLS)
   и функции, через которые ходит приложение. Секретный ключ (`service_role`/`secret`)
   тут не участвует и в приложении нигде не хранится — оно ходит только через
   публичный (`publishable`/`anon`) ключ.
3. Запусти `npm install`, затем `npm start` — приложение само создаст анонимную "учётку"
   для этого компьютера. В «Профиле» сначала предложит придумать ник (3–20 символов,
   латиница/цифры/`_`) — без него тебя не смогут найти другие.

Чтобы добавить друга: попроси его ник (или дай свой) и введи в поиск «Найти друзей» —
результаты появляются на лету, рядом с найденным — кнопка «Отправить заявку». Дальше
всё идёт через сервер: заявка, принятие/отклонение и сообщения приходят собеседнику
в реальном времени, даже если он не на вкладке «Профиль» (лишь бы приложение было
запущено).

**Регистрация** (необязательно, но желательно): в «Профиле» есть блок «Аккаунт» —
почта + пароль и кнопка «Зарегистрироваться». Это не создаёт нового человека, а
"дообустраивает" уже существующий анонимный профиль постоянным логином — ник, друзья
и переписка остаются те же. В зависимости от настроек твоего проекта Supabase может
понадобиться подтвердить почту по ссылке из письма, прежде чем вход по паролю заработает
(Authentication → Settings → там же можно и отключить обязательное подтверждение, если
это личный проект и почта не так важна). Чтобы попасть в тот же аккаунт с другого
компьютера — там в «Профиле» будет форма «Войти в существующий аккаунт» (та же почта
и пароль).

Если что-то не так, раздел «Профиль» покажет причину и кнопку «Повторить подключение» —
самые частые причины: не включены Anonymous Sign-Ins, не выполнен (или не пере-выполнен
после обновления) `supabase_schema.sql`, или нет интернета.

## Собрать в нормальный .exe с иконкой (чтобы не заходить в VS Code каждый раз)

```bash
npm run dist
```

Готовый установщик появится в папке `dist/` (на Windows — `.exe`, ставится как обычная
программа, с иконкой и ярлыком на рабочем столе). Дальше просто запускаешь его как любое
другое приложение.

## Автообновление — чтобы не пересылать .exe друзьям вручную каждый раз

Приложение умеет само проверять при запуске, нет ли версии новее, и само её ставить —
через GitHub Releases (бесплатно). Разовая настройка (10 минут):

1. **Заведи GitHub-аккаунт**, если его ещё нет (github.com → Sign up).
2. **Создай репозиторий** — New repository → назови `hanko` → можно **Public** (проще:
   не нужен токен на стороне друзей, обновления скачиваются анонимно; секретов в коде
   нет — ключ Supabase внутри специально "публичный", как объяснялось выше) → Create.
   Заливать сам код в него не обязательно — репозиторий нужен только как место, куда
   `electron-builder` будет складывать готовые установщики (Releases).
3. **Впиши свой ник** в `package.json` → `build.publish.owner` (сейчас там
   заглушка `"ЗАМЕНИ-НА-СВОЙ-GITHUB-НИК"`) и, если назвал репозиторий не `hanko`, —
   поправь `repo`.
4. **Получи токен**: GitHub → фото профиля → Settings → Developer settings →
   Personal access tokens → Tokens (classic) → Generate new token → отметь галочку
   `repo` → Generate → скопируй токен (он покажется один раз).
5. В терминале, в папке `hanko`, перед публикацией задай токен переменной окружения:
   - PowerShell: `$env:GH_TOKEN="вставь_токен_сюда"`
   - cmd.exe: `set GH_TOKEN=вставь_токен_сюда`
   (действует только для текущего окна терминала — после закрытия его нужно будет
   ввести заново перед следующей публикацией; либо один раз добавить в переменные
   среды Windows насовсем, если не хочешь повторять).

Дальше рабочий процесс для каждого обновления такой:

1. Меняешь код (в том числе через меня).
2. В `package.json` **поднимаешь номер `"version"`** (например, с `1.1.0` на `1.1.1`) —
   именно по нему приложения у друзей понимают, что вышло что-то новее.
3. `npm install` (если добавлялись новые зависимости), затем:
   ```bash
   npm run publish
   ```
   Это соберёт `.exe` и само зальёт его в GitHub Releases твоего репозитория.
4. Готово — у всех, кто уже поставил Hanko хотя бы раз, при следующем запуске
   приложение само заметит новую версию, скачает её в фоне и предложит
   «Перезапустить сейчас / Позже».

Важно: **самый первый .exe** (ту версию, с которой человек вообще ставит Hanko)
всё равно нужно передать вручную (AirDrop/диск/мессенджер, как угодно) — автообновление
работает только для версий *после* уже установленной. Также стоит один раз самому
запустить свежесобранный установщик у себя, прежде чем публиковать, — просто чтобы
убедиться, что всё открывается. И ещё: раз установщик не подписан цифровой подписью
(это платная штука), Windows/антивирус может показать предупреждение «Неизвестный
издатель» — это ожидаемо, у друзей нужно будет нажать «Всё равно запустить» / «Подробнее».

## Где хранятся данные

- Windows: `%APPDATA%\hanko\`
- Внутри: `settings.json`, `library.json` (твоя библиотека манги, прогресс чтения,
  заметки и комментарии по тайтлам), `sites.json` (сохранённые сайты для аниме +
  заметки), `online-session.json` (техническая "учётка" для друзей/чата — не трогай
  руками).
- Сами сообщения, ник, список друзей и логин/пароль хранятся не на диске, а в твоём
  проекте Supabase. Если только пользуешься анонимным профилем (без регистрации) — это
  привязано к конкретному компьютеру (в `online-session.json`). Если зарегистрировался
  (почта+пароль) — можно войти в тот же аккаунт с другого компьютера через форму «Войти»
  в «Профиле», и ник/друзья/переписка будут на месте.

Комментарии и заметки под тайтлами — только локальные (видны только тебе, не другу).
Всё остальное (манга, аниме, аватар/имя/о себе) — тоже только на этом компьютере,
никакого облака.

## Поменять название и иконку

- Название окна/установщика: `productName` в `package.json`.
- Иконка: замени `assets/icon.png` (512×512) и `assets/icon.ico` на свои — можно
  просто перегенерировать через `python3 make_icon.py`, поменяв цвета в начале файла,
  либо подставить готовый файл.

## Известные ограничения

- Раздел «Манга» тянет обложки/главы только с MangaDex — там много тайтлов, но не все,
  и это в основном фанатские переводы, а не официальные лицензионные сканы. Названия
  показываются на русском, если он есть у MangaDex (в основном названии или среди
  альтернативных) — иначе на английском. Главы по умолчанию показываются на русском;
  если для тайтла есть перевод и на другом языке (обычно английский), над списком глав
  появляется переключатель RU/EN.
- Раздел «Аниме» не парсит и не скачивает видео — это просто встроенный браузер с твоими
  закладками. Что показывает конкретный сайт и как — зависит от него самого, приложение
  в это не вмешивается.
- У каждого тайтла теперь можно оставлять личные заметку и комментарии (видны только
  тебе — не публикуются и не видны друзьям).
- «Друзья и чат» — простой личный мессенджер (текст, без вложений/картинок/звонков,
  без индикатора "печатает", без push-уведомлений, если приложение закрыто). Для двух
  человек, которые просто хотят списаться, — этого достаточно; для большего понадобится
  уже отдельный проект.
